# Phase 7. 운영 통계 상세 스펙

## 1. 개요

Phase 7은 단일 매장 오너가 예약 운영 성과를 숫자로 확인할 수 있게 하는 통계 기능이다. 목적은 정산 자동화가 아니라, 예약 수요, 방문 전환, 취소, 노쇼, 예약금/환불 흐름을 파악해 운영 결정을 돕는 것이다.

이 Phase는 Phase 1-6에서 생성된 매장, 예약, 결제, 환불, 고객 데이터를 읽어 집계한다. 멀티지점 비교, 자동 정산, 세금계산서 자동 발행, 광고 성과 분석은 범위에서 제외한다.

## 2. 목표와 비목표

### 2.1 목표

- 매장 단위 예약 성과 대시보드를 제공한다.
- 예약, 방문 완료, 취소, 노쇼 지표를 기간별로 확인한다.
- 예약금 매출과 환불 금액을 운영 참고용으로 확인한다.
- 시간대별 예약률과 상품별 성과를 파악한다.
- 필요한 범위의 통계를 CSV로 내보낼 수 있다.

### 2.2 비목표

- 정산 자동 지급은 제공하지 않는다.
- 세금계산서/현금영수증 자동 발행은 제공하지 않는다.
- 멀티지점 통합 리포트는 제공하지 않는다.
- 지도, 검색, 웨이팅 지표는 제공하지 않는다.
- 광고/쿠폰/마케팅 자동화 성과 분석은 제공하지 않는다.

## 3. 사용자 역할

| 역할 | 권한 |
| --- | --- |
| 매장 오너 | 본인 매장의 통계 조회, CSV 다운로드 |
| 플랫폼 관리자 | 모든 매장의 통계 조회, 장애/분쟁 대응용 원천 데이터 확인 |

MVP 범위에서는 직원 권한이 없으므로 매장 내부 세부 권한 분리는 제공하지 않는다.

## 4. 통계 활용 시나리오

### 4.1 매장 오너가 주간 예약 성과를 확인

1. 오너가 관리자 화면의 통계 메뉴에 진입한다.
2. 기간을 최근 7일로 선택한다.
3. 예약 수, 방문 완료 수, 취소 수, 노쇼 수를 확인한다.
4. 시간대별 예약률을 보고 한산한 시간대의 예약 상품 조정을 검토한다.

### 4.2 예약금/환불 흐름 확인

1. 오너가 이번 달 기간을 선택한다.
2. 예약금 결제 금액, 환불 금액, 순 예약금 금액을 확인한다.
3. 환불 건수가 많은 상품 또는 시간대를 확인한다.
4. 필요하면 원천 예약 목록으로 이동해 상세 건을 확인한다.

### 4.3 플랫폼 관리자가 분쟁 대응용 지표 확인

1. 플랫폼 관리자가 특정 매장을 선택한다.
2. 해당 예약의 결제/환불 상태를 확인한다.
3. 통계 수치와 원천 데이터를 비교해 고객/매장 문의에 대응한다.

## 5. 기능 상세

### 5.1 통계 대시보드

기본 카드:

- 총 예약 수
- 확정 예약 수
- 방문 완료 수
- 고객 취소 수
- 매장 취소 수
- 노쇼 수
- 예약금 결제 금액
- 환불 금액
- 순 예약금 금액

각 카드에는 현재 기간 값과 이전 동일 기간 대비 증감률을 표시한다.

### 5.2 기간 필터

기본 프리셋:

- 오늘
- 어제
- 최근 7일
- 최근 30일
- 이번 달
- 지난 달
- 사용자 지정 기간

기간 기준:

- 예약 지표는 기본적으로 `reservation_date` 기준으로 집계한다.
- 결제 지표는 `paid_at` 기준으로 집계한다.
- 환불 지표는 `refunded_at` 기준으로 집계한다.

서로 다른 기준이 섞일 수 있으므로 화면에는 "방문일 기준" 또는 "처리일 기준" 라벨을 표시한다.

### 5.3 예약 지표

집계 항목:

- 전체 예약 수
- 확정 예약 수
- 방문 완료 수
- 취소 수
- 노쇼 수
- 예약 변경 수

취소 수는 고객 취소와 매장 취소로 분리한다.

### 5.4 전환 지표

계산 항목:

- 방문 완료율 = 방문 완료 수 / 확정 예약 수
- 취소율 = 취소 수 / 전체 예약 수
- 노쇼율 = 노쇼 수 / 확정 예약 수

분모가 0인 경우 비율은 `0%`로 표시하고, 툴팁에서 "계산할 예약이 없습니다"를 표시한다.

### 5.5 결제/환불 지표

집계 항목:

- 예약금 결제 금액
- 전액 선결제 금액
- 카드 보증 건수
- 환불 금액
- 환불 건수
- 순 예약금 금액 = 결제 완료 금액 - 환불 완료 금액

주의:

- 이 지표는 운영 참고용이다.
- 정산 대상 금액과 반드시 일치한다고 보장하지 않는다.
- 정산 자동화와 세금 처리는 MVP 범위에서 제외한다.

### 5.6 시간대별 예약률

시간대 단위:

- 기본 30분 단위
- 매장 설정에 따라 60분 단위 가능

표시 항목:

- 시간대
- 예약 가능 수량
- 예약된 수량
- 예약률

계산식:

```text
예약률 = 예약된 수량 / 예약 가능 수량
```

Phase 5 좌석/재고 관리가 적용된 매장은 타임슬롯 재고 기준으로 계산한다. Phase 5가 적용되지 않은 매장은 예약 상품의 단순 재고 기준으로 계산한다.

### 5.7 상품별 성과

상품 단위 지표:

- 예약 수
- 방문 완료 수
- 취소 수
- 노쇼 수
- 결제 금액
- 환불 금액
- 평균 인원

상품 예시:

- 일반 좌석 예약
- 런치 코스
- 디너 코스
- 오마카세
- 룸 예약

### 5.8 CSV 내보내기

지원 내보내기:

- 예약 요약 CSV
- 결제/환불 요약 CSV
- 상품별 성과 CSV
- 시간대별 예약률 CSV

개인정보 제한:

- 기본 CSV에는 고객 전화번호 전체를 포함하지 않는다.
- 플랫폼 관리자용 분쟁 대응 다운로드는 별도 권한과 감사 로그를 요구한다.

## 8. API 초안

### 8.1 대시보드 요약 조회

```http
GET /owner/restaurants/{restaurantId}/analytics/summary?from=2026-05-01&to=2026-05-31
```

응답 요약:

```json
{
  "period": {
    "from": "2026-05-01",
    "to": "2026-05-31"
  },
  "reservations": {
    "total": 120,
    "confirmed": 90,
    "completed": 80,
    "cancelledByCustomer": 20,
    "cancelledByRestaurant": 5,
    "noShow": 5
  },
  "payments": {
    "depositAmount": 1200000,
    "prepaidAmount": 800000,
    "refundAmount": 300000,
    "netAmount": 1700000
  }
}
```

## 9. 지표 정의와 계산식

| 지표 | 계산식 |
| --- | --- |
| 전체 예약 수 | 기간 내 생성 또는 방문일 기준 예약 수 |
| 확정 예약 수 | `status = confirmed` 수 |
| 방문 완료 수 | `status = completed` 수 |
| 취소 수 | `cancelled_by_customer + cancelled_by_restaurant` 수 |
| 노쇼 수 | `status = no_show` 수 |
| 방문 완료율 | 방문 완료 수 / 확정 예약 수 |
| 취소율 | 취소 수 / 전체 예약 수 |
| 노쇼율 | 노쇼 수 / 확정 예약 수 |
| 순 예약금 금액 | 결제 완료 금액 - 환불 완료 금액 |
| 시간대별 예약률 | 예약된 수량 / 예약 가능 수량 |

정책 결정 필요:

- 예약 수 기준을 생성일 기준으로 볼지 방문일 기준으로 볼지 화면별 기본값을 확정해야 한다.
- 부분 환불, 카드 보증 수수료, PG 수수료를 통계에 포함할지 결정해야 한다.

## 10. 데이터 신선도와 집계 주기

기본 정책:

- 대시보드 요약은 5분 이내 지연 허용
- 당일 예약 리스트 기반 카운트는 실시간 조회 허용
- 월간/상품별 통계는 집계 테이블 사용 가능

집계 방식:

- 초기 구현은 원천 테이블 쿼리로 시작한다.
- 데이터가 증가하면 일별 집계 테이블을 도입한다.
- 결제/환불 웹훅 처리 후 관련 집계를 갱신한다.

## 11. 권한/개인정보 제한

- 매장 오너는 본인 매장의 통계만 조회할 수 있다.
- 고객 전화번호, 이메일, 상세 요청사항은 통계 화면에 노출하지 않는다.
- CSV 다운로드는 감사 로그를 남긴다.
- 플랫폼 관리자의 전체 매장 통계 조회도 감사 로그 대상이다.

## 12. 유효성 검증과 예외 케이스

### 12.1 유효성 검증

- 조회 기간의 시작일은 종료일보다 늦을 수 없다.
- 사용자 지정 기간은 기본 최대 1년으로 제한한다.
- 매장 오너는 다른 매장 ID로 조회할 수 없다.
- CSV 내보내기 타입은 허용 목록만 가능하다.

### 12.2 예외 케이스

| 상황 | 처리 |
| --- | --- |
| 기간 내 데이터 없음 | 빈 상태와 0 값 표시 |
| 환불 처리 중 | 환불 완료 금액에는 포함하지 않고 별도 상태 표시 |
| 결제 웹훅 지연 | 최근 갱신 시각 표시 |
| 집계 실패 | 원천 데이터 조회로 대체하거나 재시도 |
| CSV 생성 실패 | 실패 상태와 재시도 버튼 제공 |

## 13. 완료 기준

- 매장 오너가 기간별 예약/방문/취소/노쇼 지표를 조회할 수 있다.
- 예약금 결제 금액, 환불 금액, 순 예약금 금액을 확인할 수 있다.
- 시간대별 예약률과 상품별 성과를 확인할 수 있다.
- CSV 내보내기를 요청하고 결과를 받을 수 있다.
- 통계 조회와 CSV 다운로드에 권한 검증과 감사 로그가 적용된다.
- 정산 자동화, 멀티지점 통계, 지도/검색/웨이팅 지표가 포함되지 않는다.

## 14. 후속 Phase 연계

- 멀티지점 관리가 도입되면 지점별/브랜드별 비교 대시보드를 추가한다.
- 직원 권한이 도입되면 통계 조회/다운로드 권한을 세분화한다.
- 마케팅 기능이 도입되면 쿠폰, 재방문, 캠페인 성과 지표를 추가한다.
- 정산 자동화가 도입되면 운영 통계와 정산 리포트를 분리해 관리한다.
