# Phase 상세 스펙 목차

식당 예약 플랫폼의 Phase별 상세 스펙 문서 목록이다.

| Phase | 문서 | 목적 |
| --- | --- | --- |
| Phase 1 | [매장 입점/예약 페이지](./phase-1-store-onboarding-reservation-page.md) | 식당이 예약을 받을 준비를 완료 |
| Phase 2 | [기본 예약](./phase-2-basic-reservation.md) | 고객이 실제 예약 가능 |
| Phase 3 | [오너 예약 관리](./phase-3-owner-reservation-management.md) | 매장이 예약을 운영 가능 |
| Phase 4 | [예약금/환불](./phase-4-deposit-payment-refund.md) | 노쇼 방지와 결제 흐름 완성 |
| Phase 5 | [좌석/재고 관리](./phase-5-seat-inventory-management.md) | 시간대별 수용량을 정교하게 통제 |
| Phase 6 | [고객 관리/CRM](./phase-6-customer-crm.md) | 재방문과 리스크 관리 가능 |
| Phase 7 | [운영 통계](./phase-7-operations-analytics.md) | 매장이 예약 성과를 파악 |

전체 기획 요약은 [식당 예약 플랫폼 기획 정리](../restaurant-reservation-platform-plan.md)를 기준으로 한다.
