네이버 플레이스형 예약 UX

필터링 없이 테스트 매장 전체를 검색 결과처럼 노출합니다. 사용자는 매장명, 카테고리, 리뷰, 주소, 영업 정보와 예약 버튼을 한 카드에서 보고 진입합니다.

반영한 UX 원칙

  • 코드 입력 제거: 예약 시작점은 전체 매장 목록입니다.
  • 필터 제거: 테스트 데이터는 모두 노출하고, 정렬/필터 결정 비용을 없앴습니다.
  • 장소 카드 중심: 썸네일, 리뷰, 주소, 영업 상태, 예약 CTA를 한 번에 보여줍니다.
  • 상세 화면은 플레이스 탭 구조처럼 홈/예약/메뉴/리뷰/사진을 상단에 둡니다.

개발 적용 시 변경점

  • `/reserve`의 필터 UI와 코드 입력 폼 제거
  • 목록 API는 공개 여부와 상관없이 테스트 매장 전체를 반환하도록 조정
  • 카드 CTA는 `예약` 하나를 우선 노출하고 `/r/[slug]`로 이동
  • 상세 화면에서는 기존 예약 상품 선택 컴포넌트를 플레이스형 섹션 안에 배치
