# 사용자 FE 모바일 품질 체크리스트

## 목적

공개 예약 링크로 진입한 고객이 모바일에서 예약 생성, 조회, 취소, 결제 흐름을 안정적으로 사용할 수 있는지 릴리스 전에 확인한다.

## 대상 화면

- 공개 예약 페이지: `/r/[slug]`
- 예약 생성 플로우: 상품, 인원, 날짜, 시간, 고객 정보, 완료 카드
- 예약 조회 페이지: `/reservations`
- 예약 상세/취소 페이지: `/reservations/[reservationId]?token=...`
- 결제 후속 동선: 결제 성공, 실패, 재시도, 예약 포기

## 기준 뷰포트

| 구분            | 크기       | 확인 목적               |
| --------------- | ---------- | ----------------------- |
| Small mobile    | 360 x 740  | 좁은 Android 화면       |
| Standard mobile | 390 x 844  | iPhone 12/13/14 기본 폭 |
| Large mobile    | 430 x 932  | 큰 iPhone/Android 화면  |
| Tablet narrow   | 768 x 1024 | 태블릿 세로 모드        |

## 필수 확인 항목

| 항목           | 기준                                                                                  | 자동화        |
| -------------- | ------------------------------------------------------------------------------------- | ------------- |
| 긴 매장명      | 제목이 잘리거나 버튼을 밀어내지 않는다.                                               | `CUST-QA-001` |
| 긴 상품명/설명 | 상품 카드와 checkout summary에서 가로 overflow가 없다.                                | `CUST-QA-001` |
| 긴 주소        | 매장 정보 영역에서 줄바꿈되고 화면 밖으로 나가지 않는다.                              | `CUST-QA-001` |
| 날짜/시간 선택 | disabled 시간과 잔여 수량이 작은 화면에서 겹치지 않는다.                              | `CUST-QA-001` |
| 접근성         | `main` 영역에 axe 기본 위반이 없다.                                                   | `CUST-QA-002` |
| 지연 상태      | 가능 날짜 API가 지연되면 로딩 상태가 먼저 표시된다.                                   | `CUST-QA-003` |
| 성능           | 개발 환경 기준 LCP 5초 미만, CLS 0.1 이하를 수집한다.                                 | `CUST-QA-004` |
| 오류 추적      | API 오류는 `traceId`를 화면 메시지에 포함하고 client error report payload에 수집한다. | unit          |

## 수동 QA 메모

- 실제 결제 SDK 연동 환경에서는 결제 창 전환, 취소, 만료를 별도 단말에서 확인한다.
- 실제 매장 이미지가 들어가면 첫 화면 LCP를 다시 측정한다.
- 운영 오류 수집 endpoint가 설정된 배포 환경에서 `NEXT_PUBLIC_ERROR_REPORTING_ENDPOINT` 수신 여부를 확인한다.
